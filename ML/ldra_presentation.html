LDRA Software

Quality Assurance for Mission-Critical Software

A comprehensive software testing and verification tool for safety-critical systems

What is LDRA Software?

LDRA is a software tool that checks other software.

It helps developers make sure their programs are:

  • Correct
  • Safe
  • Secure
  • Well-tested

Primary Use Case

LDRA is mostly used for important systems where a small mistake can cause big problems (like airplanes, cars, medical machines, trains).

Why Do We Need LDRA?

Common Software Mistakes:

  • Wrong logic
  • Unused code
  • Risky instructions
  • Security problems
  • Code that is never tested

Normal Apps

Small bugs are acceptable

Critical Systems

Bugs can cause serious accidents

LDRA finds these problems early, before the software is used.

What Does LDRA Actually Do?

LDRA mainly does three things:

1. Static Analysis

Checks the code without running it

2. Dynamic Analysis

Tests the code by running it

3. Traceability

Connects requirements to code

1. Static Analysis

Checks the code without running it

How it works:

  • LDRA reads the program line by line
  • Finds errors
  • Detects bad coding style
  • Identifies unsafe instructions
  • Checks if code follows rules and standards

Example

If a rule says "don't use dangerous memory access", LDRA will warn you immediately.

2. Dynamic Analysis

Tests the code by running it

What LDRA checks:

  • LDRA runs the program with test cases
  • Which parts of code were executed
  • Which parts were never tested

This is called: Code Coverage

It answers: "Did we test everything or not?"

3. Requirements Traceability

Connects requirements to code

Every project has requirements

(what the software should do)

LDRA links:

Requirements → Code → Tests → Results

This proves that:

"Yes, this software really does what was required."

Very important for certification

Steps of LDRA Software

LDRA follows a fixed process to check software quality:

  • Step 1: Input the Source Code
  • Step 2: Static Code Analysis
  • Step 3: Code Structure Checking
  • Step 4: Standards Compliance Checking
  • Step 5: Test Case Generation
  • Step 6: Dynamic Analysis
  • Step 7: Code Coverage Checking
  • Step 8: Requirements Traceability
  • Step 9: Report Generation

Step 1: Input the Source Code

  • Developer writes the program (mostly in C / C++ / Ada)
  • This code is given to LDRA tool
Note: LDRA does not change the code, it only checks it.

Step 2: Static Code Analysis

LDRA reads the code without running it

It checks:

  • Syntax errors
  • Unused variables
  • Complex logic
  • Unsafe instructions
  • Rule violations (MISRA, CERT, etc.)
Note: This step finds problems very early.

Step 3: Code Structure Checking

This is where structure checking happens.

Control Flow

  • How the program flows
  • if-else, loops, switches
  • Detects unreachable code

Data Flow

  • How data is used
  • Uninitialized variables
  • Wrong data usage

Function Structure

  • Function length
  • Nesting depth
  • Complexity level
Note: This ensures code is clean and understandable.

Step 4: Standards Compliance

LDRA compares the code with:

  • MISRA rules
  • Safety standards
  • Company coding rules
Note: If any rule is broken → LDRA shows a warning or error.

Step 5: Test Case Generation

  • LDRA helps create test cases
  • Tests are designed to run all important paths
  • Check boundary values
Note: This reduces manual testing effort.

Step 6: Dynamic Analysis

Run the Code

  • Code is executed with test cases
  • LDRA observes program behavior
  • Runtime errors are detected
Note: This shows how the code behaves in real conditions.

Step 7: Code Coverage Checking

LDRA checks:

Statement Coverage

Lines executed

Branch Coverage

Decisions tested

Path Coverage

Logic paths

Note: Any untested code is highlighted.

Step 8: Requirements Traceability

LDRA links:

Requirements → Code → Test → Result

Note: This proves that every requirement is tested.

Step 9: Report Generation

LDRA creates:

  • Error reports
  • Coverage reports
  • Compliance reports

Used for:

  • Audits
  • Certification
  • Project review
